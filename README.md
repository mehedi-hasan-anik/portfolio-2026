# MD. Anik Miah ‚Äî Portfolio

A Next.js 15 portfolio for **MD. Anik Miah**, Software Engineer at Simec System Limited.

Built with App Router, Tailwind CSS, shadcn/ui primitives, and `motion` for animation. Optimized for Lighthouse performance and SEO out of the box, with first-class dark/light mode.

## Stack

- **Next.js 15** ‚Äî App Router, Server Components, Metadata API
- **React 19**
- **TypeScript** ‚Äî strict mode
- **Tailwind CSS** ‚Äî custom editorial theme with light + dark tokens
- **shadcn/ui** ‚Äî Button, Card, Badge primitives
- **next-themes** ‚Äî class-based theme switching with no flash on load
- **motion** (formerly framer-motion) ‚Äî page reveals, magnetic buttons, marquee, parallax
- **lucide-react** ‚Äî icons (no FontAwesome)
- **next/font** ‚Äî Instrument Serif + Geist + JetBrains Mono, self-hosted

## Design direction

**Editorial.** Two themes that share one design language:

- **Light** ‚Äî warm cream paper (`#faf3e6`) with deep charcoal text and a rust-amber accent (`#b87a1f`, AA-contrast)
- **Dark** ‚Äî deep charcoal canvas (`#0e0c0a`), warm cream text, bright golden amber (`#d4a056`)

Bold serif italics for emphasis, monospace for technical detail, a subtle film-grain overlay, ambient amber glow behind the hero and contact, magnetic buttons, a skills marquee with edge-fade mask, scroll-triggered reveals, and an interactive vertical timeline for experience.

## Theme system

Dark/light is wired through `next-themes` with class-based switching:

- `globals.css` defines both palettes ‚Äî `:root` is light, `.dark` overrides
- `<ThemeProvider>` in `app/layout.tsx` defaults to dark, respects system preference, persists to localStorage
- `<ThemeToggle>` in the navbar ‚Äî animated sun/moon swap, hydration-safe
- `viewport.themeColor` provides both `prefers-color-scheme` values for matching browser chrome

## SEO & Performance

- **Metadata API** ‚Äî `title.template`, `openGraph`, `twitter`, `robots`, `keywords`, `alternates.canonical`
- **JSON-LD** ‚Äî `Person` schema with `worksFor` (Simec System Limited), `alumniOf` (Tejgaon College), `sameAs`, `knowsAbout`
- **`sitemap.ts`** + **`robots.ts`** ‚Äî auto-generated by Next
- **`opengraph-image.tsx`** ‚Äî dynamic OG image at the edge (1200√ó630)
- **Static generation** ‚Äî every page in this project is statically generated at build
- **`generateStaticParams`** for `/blog/[slug]` so each post is pre-rendered
- **`next/font`** with `display: swap` and CSS variables ‚Äî no FOIT, no third-party font fetches at runtime
- **`next/image`-ready** ‚Äî `images.formats: ['image/avif', 'image/webp']`
- **Security headers** ‚Äî HSTS, X-Frame-Options, X-Content-Type-Options, Referrer-Policy, Permissions-Policy
- **`optimizePackageImports`** ‚Äî tree-shakes `lucide-react` and `motion`
- **Reduced-motion respect** ‚Äî `useReducedMotion` short-circuits hero and magnetic animations
- **Semantic HTML** ‚Äî `<main>`, `<header>`, `<footer>`, `<article>`, `<nav aria-label>`, `<section aria-label>`

## Getting started


The site runs at `http://localhost:3000`.

## Customization

Swap your content in **one file**: `lib/data.ts`. It owns:

- `siteConfig` ‚Äî name, role, email, phone, URL, socials
- `skills` ‚Äî marquee items
- `about` ‚Äî bio paragraphs and stats
- `experiences` ‚Äî timeline entries
- `education` ‚Äî entries
- `projects` ‚Äî work entries
- `posts` ‚Äî blog list

Update `siteConfig.url` to your real domain so the canonical, OG, and sitemap URLs all resolve correctly. The Contact form posts to a formcarry endpoint defined in `components/sections/contact.tsx` ‚Äî replace `FORMCARRY_ENDPOINT` with your own.

## Replace the blog stub

`app/blog/[slug]/page.tsx` renders placeholder body content. Wire it to your preferred source:

- **MDX in repo** ‚Äî `next-mdx-remote` or `contentlayer`
- **Headless CMS** ‚Äî Sanity, Notion, Contentful
- **Markdown files** ‚Äî `gray-matter` + `react-markdown`

The metadata, sitemap, and route structure are already in place.
